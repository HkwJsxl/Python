# 线程

### 线程理论

-   什么是线程

    ~~~
    进程：资源单位
    线程：执行单位
        将操作系统比喻成一个方的工厂
        那么进程就相当于工厂里面的车间
        而线程就是车间里面的流水线
        每一个进程肯定自带一个线程
    总结：
        进程：资源单位（起一个进程仅仅只是在内存空间中开辟一块独立的空间）
        线程：执行单位（真正被cpu执行的其实是进程里面的线程，线程指的就是代码的执行过
        程，执行代码中所需要使用到的资源都找所在的进程索要）
        进程和线程都是虚拟单位，只是为了我们更加方便的描述问题
    ~~~

    

-   为什么要有线程

    ~~~
    开设进程
        1.申请内存空间
            耗资源
        2.“拷贝代码”
            耗资源
    开线程
    	一个进程内可以开设多个线程，在用一个进程内开设多个线程无需再次申请内存空间操作
    总结：
        开设线程的开销要远远的小于进程的开销
        同一个进程下的多个线程数据是共享的！！！
        
    我们要开发一款文本编辑器
        获取用户输入的功能
        实时展示到屏幕的功能
        自动保存到硬盘的功能
    针对上面这三个功能，开设进程还是线程合适？？？
    	开三个线程处理上面的三个功能更加的合理
    ~~~

    

-   如何使用

    ​		创建进程

    ~~~python
    """demo1"""
    
    import time
    from threading import Thread
    
    
    def func(name):
        print('%s is running' % name)
        time.sleep(1)
        print('%s is over' % name)
    
    
    if __name__ == '__main__':
        t = Thread(target=func, args=('Jon',))
        t.start()
    
    ~~~

    ```python
    """demo2"""
    
    import time
    from threading import Thread
    
    
    class MyThread(Thread):
        def __init__(self, name):
            super().__init__()
            self.name = name
    
        def run(self):
            print('%s is running' % self.name)
            time.sleep(1)
            print('%s is over' % self.name)
    
    
    if __name__ == '__main__':
        t = MyThread('Jon')
        t.start()
    
    ```

### TCP并发

~~~python
"""server"""

import socket
from threading import Thread

server = socket.socket()
server.bind(('127.0.0.1', 6789))
server.listen(5)


def talk(conn):
    while True:
        try:
            data = conn.recv(1024)
            if not data: break
            conn.send(data.upper())
            print(data.decode('utf-8'))
        except Exception as e:
            break
    conn.close()


while True:
    conn, addr = server.accept()
    t = Thread(target=talk, args=(conn,))
    t.start()

~~~

### join方法

```python
import time
from threading import Thread


def func(name):
    print('%s is running' % name)
    time.sleep(1)
    print('%s is over' % name)


if __name__ == '__main__':
    t = Thread(target=func, args=('Jon',))
    t.start()
    t.join()  # 等待子进程技术后再向下运行代码
    print('main')

```

"""

current_thread()  查看当前线程信息
current_thread().name  查看当前线程名称
active_count  查看当前存活的线程个数

"""
from threading import Thread, current_thread, active_count
import os, time


def task():
    print('task pid %s' % os.getpid())
    print('task ppid %s' % os.getppid())
    time.sleep(1)
    print(current_thread())
    print(current_thread().name)


if __name__ == '__main__':
    t = Thread(target=task, )
    t.start()
    print('main')
    print('main pid %s' % os.getpid())
    print('main ppid %s' % os.getppid())
    print('存活 %s' % active_count())

### 守护线程

**主线程运行结束后，不会立即结束，要等待所有非守护子线程结束**

```python
# 守护线程 与主线程共生共死

from threading import Thread
import time


def func():
    print('start 123')
    time.sleep(1)
    print('end 123')


def foo():
    print('start 456')
    time.sleep(2)
    print('end 456')


if __name__ == '__main__':
    t = Thread(target=func, )
    t1 = Thread(target=foo, )
    t.daemon = True
    t.start()
    t1.start()  # 要等待非守护线程t1 结束，主线程才能结束
    print('main')

```

### 线程互斥锁

**将并发变成串行，牺牲效率，保证数据的安全性**

降低繁衍效率，提高血缘关系

```python
import time
from threading import Thread, Lock

money = 100


def func():
    global money
    mutex.acquire()  # with mutex:
    temp = money
    time.sleep(0.01)  # 网络延迟
    money = temp - 1
    mutex.release()


if __name__ == '__main__':
    mutex = Lock()
    lst = []
    for i in range(100):
        t = Thread(target=func, )
        t.start()
        lst.append(t)
    for li in lst:
        li.join()
    print(money)

```

### 同一个进程下的多线程无法利用多核优势，是不是就没有用了

计算密集型：

​		多进程好一些

IO密集型：

​		多线程更加节省资源

```python
# 计算密集型

from multiprocessing import Process
from threading import Thread
import time
import os


def func():
    x = 1
    for i in range(1, 100000):
        x *= i


if __name__ == '__main__':
    print(os.cpu_count())  # 4
    start_time = time.time()
    lst = []
    for p in range(5):
        # p = Process(target=func, )
        p = Thread(target=func, )
        p.start()
        lst.append(p)
    for li in lst:
        li.join()
    end_time = time.time() - start_time
    print(end_time)
# 4.060418605804443 进程
# 11.40234923362732 线程

```

```python
# IO密集型

from multiprocessing import Process
from threading import Thread
import time
import os


def func():
    time.sleep(2)


if __name__ == '__main__':
    print(os.cpu_count())  # 4
    start_time = time.time()
    lst = []
    for p in range(100):
        p = Process(target=func, )
        # p = Thread(target=func, )
        p.start()
        lst.append(p)
    for li in lst:
        li.join()
    end_time = time.time() - start_time
    print(end_time)
# 14.416825771331787 进程
# 2.0160951614379883 线程

```



### 总结

多进程和多线程都有各自的优势
		并且我们后面在写项目的时候通常可以
		多进程下面再开设多线程
		这样的话既可以利用多核也可以减少资源消耗
